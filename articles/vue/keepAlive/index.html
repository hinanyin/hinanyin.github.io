Vue 源码解析 keep-alive

发表于2025-02-28|更新于2025-02-28|前端Vue
|总字数:1.4k|阅读时长:5分钟|浏览量:|评论数:

用法

属性

  • include: 字符串或正则表达式。只有名称匹配的组件会被缓存
  • exclude: 字符串或正则表达式。任何名称匹配的组件都不会被缓存
  • max: 数字。最多可以缓存多少组件实例

规则

  • 匹配时首先检查组件自身的 name 选项，如果 name 选项不可用，则匹配它的局部注册名称 (父组件 components 选项的键值)，也就是组件的标签值。匿名组件不能被匹配
  • max 表示最多可以缓存多少组件实例。一旦这个数字达到了，在新实例被创建之前，已缓存组件中最久没有被访问的实例会被销毁掉 (LRU)
  • <keep-alive> 只处理第一个子元素，所以一般和它搭配使用的有 component 动态组件或者是 router-view

删除缓存组件并且为什么命中缓存了需要调整组件 key 的顺序：
缓存淘汰策略 LRU (Least recently used，最近最少使用) ，如果数据最近被访问过，那么将来被访问的几率也更高

生命周期钩子

  • 组件一旦被 <keep-alive> 缓存，那么再次渲染的时候就不会执行 created、mounted 等钩子函数
  • activated 和 deactivated，执行时机是 <keep-alive> 包裹的组件激活时调用和停用时调用

LRU

缓存机制

  • 将新数据从尾部插入到 this.keys 中
  • 每当缓存命中 (即缓存数据被访问)，则将数据移到 this.keys 的尾部
  • 当 this.keys 满的时候，将头部的数据丢弃
